# 板块筛选逻辑更新

## 更新日期
2026-01-22

## 更新内容

### 修改的文件
- `src/analysis/sector_filter.py` - `_select_top_sectors()` 方法

### 新的筛选规则

原有逻辑会在板块不足时从低强度板块中补齐，现在改为更严格的筛选规则：

1. **优先选择中等以上强度的板块**
   - 选择所有强度分数 >= `medium_threshold` 的板块

2. **如果中等以上强度板块不足目标数量（默认7个）**
   - 返回所有中等以上强度的板块（不补齐）
   - 例如：只有5个中等以上强度的板块，则返回这5个

3. **如果没有中等以上强度的板块**
   - 返回空列表
   - 表示当前市场没有符合条件的强势板块

4. **如果中等以上强度板块超过目标数量**
   - 按强度分数降序排序，返回前N个（默认前7个）

### 配置参数

在 `config/config.yaml` 中可以配置：

```yaml
analysis:
  sector_filter:
    high_strength_threshold: 8000  # 高强度阈值（累计涨停数）
    medium_strength_threshold: 2000  # 中等强度阈值
    target_sector_count: 7  # 目标板块数量
    ndays: 7  # N日强度统计天数
```

### 影响范围

此修改会影响：
- `ThemeAnchorAgent.analyze()` - 可能返回少于7个目标板块
- `ReportGenerator` - 需要处理目标板块数量可变的情况
- 后续的分析步骤 - 需要处理空板块列表的情况

### 优势

1. **更严格的质量控制**
   - 只选择真正有强度的板块，避免弱势板块混入

2. **更真实的市场反映**
   - 当市场整体疲弱时，返回空列表或少量板块
   - 避免强行凑数导致的误导性分析

3. **更灵活的适应性**
   - 根据实际市场情况动态调整板块数量
   - 不会因为固定数量而降低筛选标准

### 注意事项

1. **空板块处理**
   - 后续分析步骤需要处理 `target_sectors` 为空的情况
   - 建议在 `ThemeAnchorAgent.analyze()` 中添加早期返回逻辑

2. **报告生成**
   - 报告模板需要适应可变数量的板块
   - 当板块数量 < 7 时，应该在报告中说明原因

3. **用户预期**
   - 用户可能期望总是看到7个板块
   - 需要在文档中说明新的筛选逻辑
