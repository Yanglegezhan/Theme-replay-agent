## 2. 连板梯队逻辑错误 ✅ 全面修复

### 问题描述
- "3天2板" 被错误归类为"2连板"
- "8天5板" 被错误归类为"5连板"
- 实际上这些都是反包板，不是连续的连板

### 修复性质
**全面修复** - 所有"X天Y板"格式的数据都会正确处理

### 修复内容
**修改文件**：`src/data/kaipanla_source.py`

**修改函数**：`_get_consecutive_board_number`

**修改前**（错误处理所有"X天Y板"）：
```python
elif '板' in consecutive_str and '天' in consecutive_str:
    match = re.search(r'(\d+)天(\d+)板', consecutive_str)
    if match:
        return int(match.group(2))  # ❌ 直接返回板数，不判断是否是反包
```

**修改后**（正确处理反包板）：
```python
elif '板' in consecutive_str and '天' in consecutive_str:
    match = re.search(r'(\d+)天(\d+)板', consecutive_str)
    if match:
        days = int(match.group(1))
        boards = int(match.group(2))
        # 检查是否为反包板：天数 > 板数（非连续涨停）
        if days > boards:
            return -1  # ✅ 使用-1作为反包板的特殊标记
        return boards
```

### 分类逻辑（适用于所有数据）
| 原始数据 | 识别结果 | 板数返回值 | 说明 |
|---------|---------|-----------|------|
| 2连板 | 连板 | 2 | 连续2天涨停 |
| 3连板 | 连板 | 3 | 连续3天涨停 |
| 3天2板 | 反包板 | -1 | 3天内涨停2次，非连续 |
| 8天5板 | 反包板 | -1 | 8天内涨停5次，非连续 |
| 首板 | 首板 | 1 | 今天首次涨停 |

### 为什么这是全面修复？
- ✅ 所有"X天Y板"格式的数据都会经过反包板判断
- ✅ 所有"X连板"格式的数据仍然正确识别为连板
- ✅ 无需针对特定板块或特定格式做特殊处理

## 3. 容量阈值过高问题 ✅ 全面修复

### 问题描述
- 所有板块都被判定为"小容量"
- 原阈值30亿元过高，导致分类失效

### 修复性质
**全面修复** - 所有板块都会根据成交额正确分类

### 修复内容
**修改文件**：`src/analysis/capacity_profiler.py`

**修改函数**：`_classify_capacity`

**修改前**（二分类，阈值过高）：
```python
if top1_turnover > self.large_cap_turnover_threshold and has_large_cap:
    return "LARGE_CAP"  # 阈值30亿，太难达到
else:
    return "SMALL_CAP"  # 几乎所有板块都落入这里
```

**修改后**（三分类，合理阈值）：
```python
# 大容量主线：核心中军成交额 >= 15亿 且有大市值股票(>100亿)
if top1_turnover >= 15 and has_large_cap:
    return "LARGE_CAP"

# 中容量题材：核心中军成交额 5-15亿，或有大市值股票且板块总成交>=50亿
elif top1_turnover >= 5 or (has_large_cap and turnover_data.sector_turnover >= 50):
    return "MEDIUM_CAP"

# 小容量投机：核心中军成交额 < 5亿
else:
    return "SMALL_CAP"
```

### 分类标准（适用于所有板块）
| 容量类型 | 阈值标准 | 持续性评分基础分 |
|---------|---------|----------------|
| 大容量主线 | 核心中军 >= 15亿 + 大市值股 | 60分 |
| 中容量题材 | 核心中军 5-15亿 或 板块成交>=50亿 | 50分 |
| 小容量投机 | 核心中军 < 5亿 | 40分 |

### 为什么这是全面修复？
- ✅ 所有板块都会根据成交额数据自动分类
- ✅ 分类逻辑统一，无需针对特定板块调整
- ✅ 阈值合理，符合A股市场实际情况
